## 3. 登入與認證流程（Login / Auth Flow）

| 項目 | 現狀 | 決策 |
|------|------|------|
| 認證方式 | Bearer token，由互動式登入流程自動管理 | **公開文件描述高層次互動式按需登入流程** |
| 互動式登入流程 | README 已說明：工具會在需要時自動觸發登入，token 由登入流程自動管理 | **已公開描述高層次流程；使用者無需手動設定 token** |
| 低層次 token 取得細節 | 包含反向工程端點、內部 session / cookie 機制等實作細節 | **保守預設：不在公開文件中描述低層次 token 取得步驟或反向工程端點** |

**現狀說明**：README 目前已公開描述高層次的互動式按需登入流程，並說明 token 由登入流程自動管理，使用者無需手動注入憑證。**刻意省略**的是低層次的 token 取得機制（如反向工程的認證端點、內部 session 處理邏輯），以避免引導未授權的直接 API 存取。

**待確認**：是否有合法的公開 token 取得流程（如 OAuth、公開 API key 申請）可在文件中補充說明？

## 4. 非官方聲明（Non-Official Status）

本倉庫為**非官方、社群維護**的 MCP 整合工具，與上游服務提供商無正式關聯。

- 本工具不由上游服務官方開發或背書
- 使用者應自行評估合規性與服務條款
- 維護者不對因使用本工具造成的帳號限制或服務中斷負責

**建議**：此聲明應反映於 README 中（目前尚未明確加入），並在所有公開溝通中保持一致。待 README 補充非官方、社群維護及無正式關聯等聲明後，請將本項狀態更新為已完成。

## 5. SECURITY.md vs NOTICE 檔案

| 檔案 | 現狀 | 理由 |
|------|------|------|
| `SECURITY.md` | **已新增** | 提供漏洞回報管道，符合公開倉庫最佳實踐 |
| `NOTICE` | **未新增** | 目前無需聲明第三方授權或商標歸屬 |

**決策依據**：
- `SECURITY.md` 屬於標準社群健康檔案，不涉及敏感資訊，優先新增。
- `NOTICE` 通常用於 Apache License 2.0 或需要聲明第三方元件歸屬的情境。目前採用 MIT License，且無需特別聲明的第三方元件，故暫不新增。

**待確認**：若未來引入需要 NOTICE 聲明的依賴（如 Apache 2.0 授權元件），需補充 `NOTICE` 檔案。

## 6. 測試與文件邊界（Testing / Documentation Boundary）

### 測試策略

| 類型 | 現狀 | 決策 |
|------|------|------|
| 預設測試（`pytest`） | 離線單元測試，使用 mock | **必須可離線執行，不依賴真實 API** |
| 整合測試 | `tests/test_all_tools.py`，需真實 API | **opt-in，需設定環境變數 `BUY123_RUN_LIVE_TESTS=1`** |
| CI 流程 | GitHub Actions，僅執行離線測試 | 不在 CI 中執行整合測試 |

**保守預設**：任何需要真實憑證或網路連線的測試，必須明確標記為 opt-in，不得成為預設測試流程的一部分。

### 文件邊界

| 項目 | 決策 |
|------|------|
| 私有 API 文件 | **不納入倉庫** |
| 內部操作手冊 | **不納入倉庫** |
| 環境設定範例 | 僅提供變數名稱，不提供真實值或範例憑證 |
| `.env.example` | 若新增，僅包含變數名稱與說明，值留空 |
